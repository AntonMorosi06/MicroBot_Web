System Structure

Layered Architecture

The MicroBot system is organized as a modular architecture in which simulation, visualization, and physical execution remain separated but conceptually aligned.

Simulation Layer

Models local rules, agent interactions, and emergent collective behavior in a controlled software environment.

Web Layer

Exposes structure, metrics, and visual system behavior through a browser-based interface designed for interpretation and presentation.

Hardware Layer

Translates simplified swarm logic into physical outputs through an ESP32-based LED prototype.

Simulation Module

Behavioral Logic

The simulation module provides the reference implementation of the system’s swarm logic through local interaction rules such as alignment, cohesion, and separation.

Core Features

  • Multi-agent environment
  • Local interaction rules
  • Mode switching
  • Real-time visual feedback

Current Modes

  • Alignment
  • Flocking
  • Disorder

Hardware Module

Physical Prototype

The current hardware prototype uses a single ESP32 and multiple LEDs to represent simplified node states and coordinated patterns.

Current Setup

One ESP32 board controls five LED-based nodes through digital GPIO outputs and serial-triggered mode switching.

Future Direction

The long-term goal is to move toward independent nodes, local communication, sensing, and physically distributed execution.
